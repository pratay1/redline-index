# Redline Index

## Register

brand

Public catalogue surfaces are brand-led: the design *is* the product experience. Tool density appears on browse/search/detail, but the default voice is editorial automotive platform—not admin SaaS.

## Platform

web

## Users

**Primary:** Automotive enthusiasts browsing verified specs for curiosity, comparison, and deep product knowledge.

**Secondary:** Researchers and buyers validating trim-level data and sources.

## Purpose

Preserve, search, and publish verified vehicle records with clear provenance—so enthusiasts can trust every number.

## Positioning

The precise public index of verified machines: manufacturer lineage → generation → year → trim, with sources attached.

## Brand personality

Precise. Archival. Premium.

## Voice

Short, technical, confident. Prefer “record,” “index,” “trim,” “verified” over marketing fluff.

## References

- Apple — typography scale, whitespace, product photography hierarchy
- Porsche / Rivian — automotive materiality and restrained luxury
- Linear / Stripe / Vercel — clarity, craft, calm dark UI systems

## Anti-references

- Gaming landing pages, neon cyberpunk, heavy mesh/gradient stacks
- Generic AI SaaS (purple glows, three equal feature cards, glass everywhere)
- Dashboard chrome and dense admin tables on public marketing surfaces

## Conversion & proof

- **Primary CTA:** Search / open a verified record
- **Secondary:** Browse manufacturers or catalogue
- **10-second memory:** “Machines, indexed.”
- **Belief ladder:** This is authoritative → records are verified → I can find the exact trim → sources back the numbers

## Accessibility

WCAG AA target. Respect `prefers-reduced-motion`. Visible focus rings. Semantic headings. Touch targets ≥44px where interactive.

## Design principles

1. Photography and typography carry the brand; color is a signal, not decoration.
2. One job per section; generous rhythm between sections, tight grouping within.
3. Motion is purposeful and subtle—entrances, hover, stagger—not spectacle.
4. Prefer hairline structure and tonal surfaces over cards and gradients.
5. Reuse components; never invent one-off chrome for the same pattern.
